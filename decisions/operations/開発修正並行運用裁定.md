# 開発修正並行運用裁定

## 裁定

- 裁定: 会議進行と並行して `company-os` 自身の開発修正を常設運用する。
- 理由: 制度改善を実装に接続し、運用負荷を下げながら学習速度を上げるため。
- 少数意見: なし
- 再審条件: 新事実または手続瑕疵
- 実行責任者: `engineering`, `quality-assurance`, `release-operations`, `internal-tools-enablement`, `project-management-office`
- 創業者拒否権: なし

## 状態更新

- 開発修正ループを正式運用に切り替えた
- 開発修正台帳を開いた
- Engineering / QA / Release / Internal Tools を実働状態に切り替える

## 次

1. 開発修正台帳の優先項目を継続更新する
2. 会議で発見した詰まりを修正台帳へ戻す
3. high risk は既存の release / approval 経路を維持する
