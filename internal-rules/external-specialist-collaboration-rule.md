# External Specialist Collaboration Rule

## Purpose

Allow Company OS to use outside specialists or parallel agents for bounded execution without giving away governance, approval authority, or organizational design control.

## Allowed use

External specialists may be used for:

- bounded artifact production
- visual or implementation exploration
- research or draft generation
- narrow execution tasks with clear acceptance criteria

## Prohibited transfer

External specialists may not:

- amend rules or charters
- reorganize departments
- exercise stop powers
- approve release or high-risk work
- become the system of record

## Sponsorship rule

Every external specialist lane must have:

1. a sponsoring lawful department
2. a PMO-tracked handoff path
3. an internal reviewer
4. an acceptance owner
5. a visible trace in the external specialist board

## Acceptance rule

External output becomes effective only after a lawful internal owner accepts, records, and routes it through the required approval path.

For bounded UI work, the sponsoring design department may serve as the acceptance owner when governance, release, and high-risk approval powers remain internal.
