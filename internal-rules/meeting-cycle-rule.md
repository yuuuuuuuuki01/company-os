# Meeting Cycle Rule

## Purpose

Make ordinary sittings run as a loop instead of isolated meetings.

## Agenda classes

The assembly may recognize three agenda classes:

- `report`: reporting only, no vote required
- `discussion`: substantive issue under shaping or coordination
- `deliberation`: issue ready for decision, vote, or ruling

## Required cycle for substantive items

Each substantive item should normally pass through:

1. `discussion-1`
2. `discussion-2`
3. `deliberation`

## Exception

The chair may compress the cycle only when urgency is recorded, prior written discussion already exists, and no higher-order review is skipped.

## Next-agenda trigger

Before every sitting closes, the assembly must:

1. confirm the next agenda batch
2. assign the next owner for each carried item
3. open or update the next ordinary sitting record

This next-agenda confirmation is the trigger that starts the following meeting loop.
