# Meeting Work Coupling Rule

## Purpose

Prevent the assembly from producing discussion-only paperwork without operational output.

## Default requirement

Each ordinary sitting should leave at least one of the following for every active substantive item:

- a rule draft
- an SOP draft
- a ledger update
- a template
- a bounded implementation artifact

## Work-first override

If a sitting does not also move active implementation or active bundles forward, new discussion items should not be opened except for narrow exception handling.

## Reporting-only exception

`report` items may avoid new artifacts only when they already carry completed work and the sitting is merely acknowledging status.

## Knowledge rule

If a meeting changes how work should be done, the change must be written into a reusable operating artifact, not only into a ruling note.
