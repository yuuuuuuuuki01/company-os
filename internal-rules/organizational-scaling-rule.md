# Organizational Scaling Rule

## Purpose

Allow Company OS to add departments, divisions, and sections as load grows, while keeping every new layer reviewable and reversible.

## Scaling principle

- scale by smallest sufficient layer first
- prefer a section before a division, and a division before a new department
- create a new top-level department only when the mandate clearly escapes existing departmental ownership
- if repeated interdepartment handoffs create durable context debt, split the context instead of forcing the parent department to absorb it

## Required fields

Every scaling request must state:

1. parent line
2. proposed layer
3. bounded mandate
4. delegated authority
5. escalation route
6. measurable overload or opportunity signal
7. rollback or merge-back condition

## Default thresholds

- section creation: internal-rule threshold unless stop powers or charter powers change
- division creation: article-level threshold
- new department: article-level threshold and growth-board review

## Delegation hygiene

- no subdivision may inherit silent authority
- approval powers, stop powers, and record duties must each be separately declared
- every active subdivision must have a named review point
