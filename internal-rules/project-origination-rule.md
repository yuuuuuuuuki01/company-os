# Project Origination Rule

## Purpose

Allow Company OS to create new project options autonomously while keeping origination separate from activation, staffing, and commitment.

## Allowed outputs

Exploration and Incubation may create:

- opportunity signals
- bounded hypotheses
- incubation briefs
- comparative option notes
- readiness-screen requests

## Minimum inputs

Every originated item must name:

1. a concrete signal source
2. a short problem statement
3. an intended beneficiary or sponsor
4. a bounded scope hypothesis
5. a next gate

## Prohibited shortcuts

The origination loop may not:

- open a new active lane directly
- commit budget or staffing outside normal approval
- use protected credentials without separate approval
- bypass Portfolio Strategy, PMO, or constitutional controls
- convert an idea into a public commitment by memo alone

## Promotion gate

An originated project may move beyond brief stage only when:

1. Portfolio Strategy accepts it as a candidate
2. PMO confirms it can be screened without overload
3. the required readiness-screen motion is filed
4. the lawful approval path is satisfied
