# スクリプト監査ルール

## 目的

導入されたスクリプトが形骸化、過剰適用、漏れ、誤適用を起こしていないかを継続監査する。

## 監査観点

- 本当に定型業務だけに適用されているか
- 例外条件や branch 条件が守られているか
- 人間確認を残すべき箇所が消えていないか
- output が台帳や記録に正しく反映されているか
- 適用漏れや監査漏れがないか

## 監査結果

- 適正
- 条件付き適正
- 是正必要
- 停止勧告

## 必須是正

- 適用範囲が曖昧な script は是正対象
- 例外条件未記載の script は是正対象
- 高リスク処理を誤って自動化した script は即時停止勧告
