# 定型業務スクリプト化ルール

## 目的

業務や議論の中で定型化した処理を、適切な境界の中で自動的にスクリプト化し、トークン消費と反復負荷を減らす。

## 基本原則

- 同じ処理が `3回以上` 反復した場合、まずスクリプト化候補として評価する
- 候補は `script-first`, `cli-first`, `template-first` の順で置換手段を検討する
- 低リスクの状態確認、台帳整流、定型変換、handoff 補充は自動スクリプト化の優先対象とする
- 高リスク、秘密情報、不可逆操作を含む処理は自動スクリプト化対象にしない

## 自動スクリプト化の条件

- 入力と出力が一定である
- 成功 / 失敗の判定が短く明示できる
- 既存ルールと台帳に接続できる
- branch 条件や例外条件を文書化できる

## 必須記録

- script id
- 対象業務
- 適用範囲
- 禁止範囲
- 監査対象
- rollback 手段

## 禁止

- 定型に見えるだけで、判断根拠が不安定なものを自動化する
- script 導入後に人間の検証線を全て消す
- 例外条件を書かずに運用へ入れる
