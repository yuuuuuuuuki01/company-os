# 承認分岐運用ルール

## 目的

承認が必要な論点で全体の進行を止めず、承認待ち論点だけを分岐させて他の作業を継続する。

## 基本原則

- 承認が必要な論点は `approval branch` として切り出す
- branch 化された論点は `承認分岐台帳` に必ず記録する
- branch と無関係な low/medium 作業は継続する
- 承認待ちの有無を理由に ordinary sitting 全体を停止しない

## branch 化の条件

次のいずれかに当たる場合は branch 化する。

- Founder 返却条件に触れる
- 新しい鍵、資格情報、秘密情報の利用が必要
- `high risk` 判断が必要
- 既存PJの範囲を超える活性化判断が必要
- 不可逆操作や履歴破壊を伴う

## branch の必須項目

- `branch_id`
- `approval_scope`
- `approver`
- `blocked_item`
- `parallelizable_items`
- `requested_at`
- `return_condition`

## 並行継続の原則

- `parallelizable_items` は owner と next handoff を持つ
- branch に直接依存しない議題は auto mode で進めてよい
- report item、tooling item、記録整備は branch 中でも carry する

## 閉鎖条件

- 承認済みで本流へ戻す
- 却下して branch を閉じる
- 代替案へ付け替えて branch を閉じる
