# 文脈分離成長ルール

## 目的

既存部署が専門外の文脈を抱え込み続けることを防ぎ、必要な部署・部門・課が自律的に増えて成長できるようにする。

## 基本原則

- 成長は `人を増やす前に文脈を分ける` を優先する
- 新しい責務が既存部署の主目的を曇らせるなら、section または division を切る
- それでも parent 部署の mandate を逸脱するなら、新部署化を提案する
- 組織成長は `top-down expansion` と `bottom-up overload signal` の両方で起こせる

## 自律増殖シグナル

- handoff の再説明コストが増えた
- 固有用語や固有判断基準が増えた
- review が parent 部署の速度を明確に落としている
- active PJ がその能力を継続的に要求している
- trainee が複数名必要になるほど役務が安定した

## 成長順序

1. section を切る
2. division へ昇格させる
3. 新部署化する

## 抑制条件

- 単発の過負荷だけでは新部署化しない
- active PJ 1件だけの一時負荷は temporary task delegation を優先する
- 新部署化には既存部署から切り離す文脈と記録資産が明確でなければならない
