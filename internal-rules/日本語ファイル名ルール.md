# 日本語ファイル名ルール

## 目的

Company OS の文書を、できるだけ日本語で読める形に寄せつつ、参照切れや運用停止を避けながら段階移行する。

## 基本方針

- 今後新しく作る制度文書、台帳、議案、裁定、手順書は原則として日本語ファイル名を使う
- 既存ファイルは一括改名しない
- 既存ファイルは更新や再編のタイミングで段階的に日本語名へ移す
- 外部仕様、既存連携、第三者規約に強く依存する名前は例外を認める

## 命名原則

- できるだけ短い日本語名にする
- 役割が一目で分かる語を使う
- 句読点や機種依存文字は避ける
- 必要なら半角ハイフンで語を補助する
- 英語固有名詞は必要最小限だけ残す

## 例外

次は英語や既存名を残してよい。

- 外部プロダクト名
- API や規格名
- GitHub Pages やツールが前提にする固定名
- 既存参照が多く、即時改名の方が危険な基幹ファイル

## 移行原則

- 改名時は参照元を同一変更で更新する
- 改名は原則として touched-file 単位で行う
- 大量一括改名は別議案で扱う
- 移行状況は `ledgers/日本語命名移行台帳.md` に記録する
