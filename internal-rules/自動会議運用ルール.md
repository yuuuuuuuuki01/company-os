# 自動会議運用ルール

## 目的

通常運転の改善議題を、できるだけ Founder の逐次確認なしで自動的に回す。

## 自動運転の対象

次の条件を満たす議題は、自動会議運転の対象にしてよい。

- `risk` が `low` または `medium`
- 憲法、法律、定款の改定を含まない
- 外部公開、秘密、鍵、資格情報、課金、削除、不可逆変更を含まない
- reusable artifact が会議ごとに残る
- owner と next handoff が明確

## 自動で進めてよい行為

- ordinary sitting の起票、更新、閉会
- report 議題の継続監視
- discussion-1 から discussion-2、deliberation への進行
- 境界内の deliberation 議題の採択
- 台帳、ルール、SOP、雛形、短報裁定の更新
- `company-os` リポジトリへの通常の commit と push

## 異議と改善案

- 議長は各 substantive item で、質問、意見、建設的改善案を必ず促す
- blocking objection が記録された場合、その議題は自動採択せず次会議へ送る

## Founder へ戻す条件

- `high risk`
- 憲法、法律、定款の改定
- 新しい秘密情報や鍵の使用
- 役職の大幅な権限変更
- 不可逆な削除や履歴破壊
- 既存4PJを超える追加PJ活性化判断

## 既定動作

- ordinary sitting は閉会時に次回議題を自動で開く
- active item は原則 3 から 7 件で束ねる
- report item は watch が継続する限り自動で carry する
