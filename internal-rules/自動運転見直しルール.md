# 自動運転見直しルール

## 目的

導入した automation が過剰運転、重複、空振り、停止点化を起こしていないかを定期的に見直す。

## 見直し観点

- 役割重複がないか
- 実行頻度が高すぎるか低すぎるか
- queue 補充、handoff 補充、branch 分離に実効性があるか
- 報告が停止点になっていないか
- 期待した台帳更新と実際の更新量が釣り合っているか

## 見直し結果の選択肢

- 維持
- 頻度変更
- prompt 縮小または拡張
- 他 automation との統合
- 停止

## 必須出力

- 見直し結果の短報
- 変更の要否
- 次回見直し時点
