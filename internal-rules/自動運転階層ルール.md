# 自動運転階層ルール

## 目的

自動運転を単一周期で回さず、`軽い監視`, `小補充`, `通常運転`, `巡回監督`, `見直し` を分けて運用する。

## 理想の階層

- `10分級`: 死活監視、handoff 空欄検知、queue 枯渇検知
- `30分級`: note / bundle / handoff の小補充
- `1時間級`: low / medium の通常運転
- `4時間級`: portfolio / branch / 停滞の巡回監督
- `8時間級`: 優先順位と運転束の見直し

## 実装制約

- 現行の automation 実装は `時間単位` を基本とする
- `10分級`, `30分級` は app automation では直接再現せず、`1時間級` の軽量チェックに折り畳む
- したがって実運用の自動化は `1時間`, `4時間`, `8時間`, `24時間` を基本単位とする

## 運用原則

- 短周期ほど軽くし、会議や重い審査は持ち込まない
- 長周期ほど portfolio, branch, automation 自体の見直しを担う
- automation の役割が重複した場合は統合か責務縮小を検討する
