# 非稼働事業審査起票ルール

## 目的

非稼働事業の `note` や `審査束` が、そのまま承認済みと誤認されることを防ぎ、必ず正式な審査起票と裁定記録を通す。

## 原則

- `note-issued` と `bundle-issued` は `review-ready` を意味するだけで、承認済みを意味しない
- 非稼働事業は、active 化しない場合でも `審査起票` と `裁定記録` を持たなければ review を通過したとはみなさない
- low / medium の審査でも、auto ordinary sitting か所管 review body への起票を必須にする

## 必須状態遷移

- `packet-issued -> note-issued -> bundle-issued -> review-opened -> review-adopted or branch-open`
- `review-opened` の時点で、議案または review request 文書への参照を持つ
- `review-adopted` の時点で、裁定文書または正式な返却記録への参照を持つ

## 審査起票の基準

- utility screening, light readiness review, boundary clarification は `medium 以下` なら自動会議運用で起票できる
- high-risk screening, policy-first screening, public / safety 文脈は所管 gate owner が review opening を明示し、必要なら承認分岐へ送る
- Founder 返却条件に触れる論点は [承認分岐運用ルール.md](C:/agent/codex/company-os-sync/internal-rules/承認分岐運用ルール.md) に従う

## 台帳要件

- [非稼働事業審査順序台帳.md](C:/agent/codex/company-os-sync/ledgers/非稼働事業審査順序台帳.md) は各事業について `review opening` と `decision record` を持つ
- `review opening` が空欄の項目は queue に載っていても未審査扱いとする
- `decision record` が空欄の項目は review 完了扱いにしない
