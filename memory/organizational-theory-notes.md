# Organizational Theory Notes

This file is the seed note for importing external academic insight into Company OS redesign work.

## Initial lenses

- Bounded rationality: keep packets and decisions small enough to be understandable.
- Principal-agent risk: align delegated authority with auditable records and review rights.
- Transaction cost logic: formalize handoffs only where coordination cost justifies the process.
- Sociotechnical design: treat tools, rules, and people as one operating system.
- Contingency logic: do not force one structure on all units if environment and risk differ.
- Metric distortion: do not let a metric replace the underlying purpose it approximates.
- Path dependence: periodically rebuild teams and procedures to avoid locked-in local optima.
- Communication structure effects: use PM and Director as anti-silo cross-links.
