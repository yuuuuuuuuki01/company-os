# オーケストレーション設計メモ 2026-04-10

## 発想転換

- 会社OSは `会話中心の運用` から `MCP / SaaS / script / 監査` のオーケストレーション層へ進む。
- 既存部署に無理に吸収させず、必要なら部門や部署を増設する。
- local LLM は `仕様化された bounded work` のみを受け取って script 草案を作る。

## ループ

`業務検知`
→ `script 候補化`
→ `仕様化`
→ `草案生成`
→ `限定実行`
→ `監視`
→ `次手生成`

## 直近の次手

1. `review packet issuance helper` を初回 script 化候補として進める
2. `script-audit sample helper` を監査線で進める
3. company-os運転スレッドは CLI-first 実装へ寄せる
4. 監査スレッドは sample の妥当性だけに集中する
