# 全体整理メモ 2026-04-05

## 現状の整理

- 会議先行の癖が強かったため、`作業先行` と `自動化実装連鎖` で補正した
- 事業は `事業責任者 + PM + Director` の triad で共同運営する形へ寄せた
- 部署間 handoff が増えたため、連携経路を可視化し、文脈が増えたら section, division, department へ分離する方針を明文化した
- trainee 採用は一部ではなく全部署・主要役所へ広げた

## いま特に重要な論点

- 連携が一時支援なのか、分化候補なのかを見極めること
- 事業 triad が unit-owner と競合せずに運営できること
- trainee の増加が単なる人海戦術でなく、文脈分離と組織成長へつながること
