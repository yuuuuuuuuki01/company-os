# Committee Execution Bridge Motion

## Goal

Tighten the handoff from committee findings into execution lanes so findings do not stall after review.

## Proposed action

- require each committee finding to name one executor and one deadline
- add a handoff field to committee outputs
- send binding findings directly to the responsible lane owner

## Why

- review authority exists, but post-review execution is still loose
- binding findings should move work, not only describe it

## Discussion-1 note

- committee findings need an executor, deadline, and handoff target
- binding findings should update the receiving lane directly
- send to `discussion-2` with a minimal output schema
