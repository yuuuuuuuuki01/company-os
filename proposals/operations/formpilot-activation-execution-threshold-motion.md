# FormPilot Activation Execution Threshold Motion

## Goal

Define the minimum threshold that must be met before any later bounded execution motion may move `FormPilot` beyond `not-activated`.

## Proposed action

- require the `oem`, `shift`, and `jouzou` active-project watch to remain free of critical exception
- require the bounded `FormPilot` seat route, packet, reviewer route, and approval path to remain current
- require PMO and Security, Risk, and Compliance to confirm that a fourth active project remains out of scope until a later explicit decision

## Why

- an activation-review path alone should not trigger execution
- fourth-project sequencing should stay slower and more bounded than the third lane

## Discussion-1 note

- this motion should define readiness thresholds only and should not activate `FormPilot`
- accepted artifact set: `ledgers/formpilot-activation-review-checklist.md`
- discussion should stay limited to the minimum lawful execution threshold

## Discussion-2 note

- execution threshold should stay limited to watch, checklist, PMO capacity, and security posture
- this motion must not authorize seat execution or PJ activation
- accepted artifact set: `ledgers/formpilot-execution-threshold-board.md`
- deliberation should decide whether the minimum threshold is narrow enough to support only later execution-bundle preparation

## Deliberation result

- adopted as the minimum lawful execution threshold for `FormPilot`
- `FormPilot` remains `not-activated`
- the next lawful step is execution-bundle preparation only
