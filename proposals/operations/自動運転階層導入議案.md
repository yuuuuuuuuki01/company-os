# 自動運転階層導入議案

## 目的

自動運転を複数周期に分け、監視、補充、通常運転、巡回監督、見直しを役割別に分ける。

## 提案

- 理想の `10分 / 30分 / 1時間 / 4時間 / 8時間` を階層として定義する
- 実装制約に合わせ、app automation では `1時間 / 4時間 / 8時間 / 24時間` を基本単位にする
- automation 自体の見直しを `24時間級` で定例化する

## 期待効果

- 過剰な高頻度運転を避けられる
- 軽い監視と重い巡回を分けられる
- automation 自体が古くならない
