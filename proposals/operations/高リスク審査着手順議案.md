# 高リスク審査着手順議案

## 目的

`credential-ledger / Poin-T / saigai / zenken` の high-risk / policy-first 項目を、無秩序に開かず gate 順で扱う。

## 提案

- `credential-ledger` を最初の high-risk screening 候補にする
- `saigai` は policy-first のため別レーンで扱い、Founder 側の例外確認を先行させる
- `Poin-T`, `zenken` は release-review-committee capacity を見ながら後続に置く
- high-risk 項目は review-ready でも approval branch 条件を先に確認する

## 期待効果

- high-risk gate の過負荷を避けられる
- policy-first と screening を混ぜずに進められる
- 24時間運転でも止めるべきものだけ止められる
