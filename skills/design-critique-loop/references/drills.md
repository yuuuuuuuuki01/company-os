# Design Drills

Run only one drill at a time.

## Drill 1: One-sentence diagnosis

- Describe the current UI failure in 20 words or fewer.
- Ban generic words like "cool", "clean", or "good".

## Drill 2: Stronger hierarchy

- Keep the same content.
- Change only spacing, scale, and emphasis.
- Make the first read path obvious.

## Drill 3: Direction fork

- Describe two distinct visual directions.
- Pick one and reject the other explicitly.

## Drill 4: Paper load audit

- Count how many competing panels, cards, or actions exist.
- Remove or collapse weak items before styling more.

## Drill 5: Reusable lesson

- Write one lesson that would help the next designer avoid the same mistake.
