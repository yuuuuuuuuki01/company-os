# Run Parallel Operations

## Purpose

Run multiple unit lanes without losing ownership, approval trace, or handoff clarity.

## Steps

1. select candidate units from `units/registry.md`
2. open or update a row in `ledgers/parallel-operations-board.md`
3. declare current work, next handoff, and paper load
4. if approval is needed, add or update a row in `ledgers/approval-desk-board.md`
5. reflect active lanes in the dashboard
6. mark stalled lanes as `blocked`, `ready`, or `idle` instead of leaving ambiguous text

## Exit Condition

A lane is considered safely operating in parallel when:

- the lane status is visible
- the next handoff is named
- approval debt is visible at the approval desk
