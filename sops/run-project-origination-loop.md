# Run Project Origination Loop

## Goal

Generate new project options in a bounded way without confusing ideation with activation.

## Steps

1. collect signals from active lanes, audit findings, academic intake, and external patterns
2. write a one-paragraph hypothesis for each viable signal
3. discard or defer ideas with no sponsor, no bounded scope, or no evidence path
4. upgrade only the strongest items into incubation briefs
5. record every item in `ledgers/project-origination-board.md`
6. hand readiness-screen candidates to Portfolio Strategy and PMO
7. keep all non-promoted items in backlog or mark them dropped

## Output standard

- reports stay in short Japanese when routed into ordinary operations
- each item must show current stage, intended sponsor, and next gate
- no item may claim activation before a lawful readiness-screen motion exists
