# SOP: Run Unit Activation Review

1. Confirm the unit is `registered`.
2. Confirm the unit has a lawful `seat_state` or an explicit bootstrap exception.
3. Confirm a mission packet exists.
4. Confirm owner, reviewer, and approval path are named.
5. Set lane state to `ready`, `blocked`, or `active`.
6. Record the result in the parallel operations board.
