# トークン節約実行ループ

## 手順

1. 対象作業が `反復`, `定型`, `集計`, `状態確認` のどれかに当たるか見る
2. 当たる場合は、既存の `CLI`, `script`, `template`, `ledger` で処理できるか確認する
3. 既存で足りなければ、低リスクなら短いCLI入口かテンプレートを追加する
4. それでも足りない場合のみ長い対話や重い会議へ送る
5. 置換した内容を部署台帳と通達台帳へ反映する

## 出力

- 短いCLI入口
- 定型テンプレート
- 台帳更新
- 置換候補の短報
