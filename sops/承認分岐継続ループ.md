# 承認分岐継続ループ

## 手順

1. branch 化条件に触れた論点を検出する
2. `承認分岐台帳` に branch を起票する
3. approver と return condition を明記する
4. blocked item と parallelizable items を分ける
5. blocked item は approval queue に送る
6. parallelizable items は ordinary sitting と実務台帳で継続する
7. approval が返ったら本流に戻すか却下で閉じる

## 継続対象

- low/medium の rule、SOP、ledger、template 更新
- report watch
- 実務上の handoff と記録整備
- branch 非依存の tooling 整備

## 停止対象

- branch 承認がないと進めない high risk 実装
- branch の結論がないと意味が確定しない不可逆作業
