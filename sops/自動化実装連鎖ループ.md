# 自動化実装連鎖ループ

## 手順

1. [自動化運用決定束台帳.md](C:/agent/codex/company-os/ledgers/自動化運用決定束台帳.md) から `queued` と `active` を確認する
2. `active` 束が `watch` へ移ったら、次順位の `queued` 束を自動で `active` に更新する
3. 空きがない場合は、次順位束を `queued` のまま維持し、会議には report だけ返す
4. 高リスク再分類や依存詰まりが起きた時だけ ordinary sitting に例外議題を上げる
5. 例外がなければ会議は `report` と `watch` 更新中心に回す

## 出力

- 束状態更新
- 次束昇格記録
- 例外議題が必要な場合だけの上申
