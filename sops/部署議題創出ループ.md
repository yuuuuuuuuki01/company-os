# SOP: 部署議題創出ループ

1. 各部署は current assignment と watch 中の摩擦から候補を `1` 件以上洗い出す。
2. 候補は [部署議題創出台帳.md](C:/agent/codex/company-os/ledgers/部署議題創出台帳.md) に `候補名 / 種別 / 根拠 / 優先帯 / 想定上申先` を記録する。
3. 候補が薄い部署には、PMO と Data and Knowledge が叩き台を返す。
4. assembly-secretariat は候補を `report / discussion / deliberation` の入口に割り振る。
5. 閉会時に、次回までの提出担当部署と不足論点を明記する。
6. 次回 sitting の起票前に、未提出部署への督促を記録する。
